## 📖 О проекте

В рамках учебной проектной работы я спроектировал и настроил **корпоративную инфраструктуру** для компании «Суперстор», которая открывает новый офис и даркстор. Проект охватывает три ключевые области:

1. **Сеть** — VLAN-изоляция отделов, маршрутизация, NAT, публикация веб-сервера, выделенный канал между площадками.
2. **Сервисы Linux** — DNS, NTP, почта (Postfix), FTP (vsftpd), веб-сервер (Nginx), управление конфигурациями (Ansible).
3. **Active Directory** — домен, организационные единицы, групповые политики, ввод клиентов в домен.

**Цель** — продемонстрировать практические навыки сетевого инженера, системного администратора и специалиста по информационной безопасности на примере реальной бизнес-задачи.

## 🛠 Стек технологий

| Категория | Технологии |
|-----------|------------|
| **Сетевое оборудование** | Cisco (маршрутизаторы, коммутаторы), эмуляция в EVE‑NG |
| **Серверные ОС** | Ubuntu 20.04 LTS, Windows Server |
| **Сетевые протоколы** | VLAN (802.1Q), OSPF (статическая маршрутизация), NAT/PAT, DHCP, DNS |
| **Сервисы Linux** | `chrony` (NTP), `dnsmasq` (DNS/DHCP), `Postfix` (почта), `vsftpd` (FTP), `Nginx` (веб), `Ansible` (автоматизация) |
| **Active Directory** | AD DS, DNS, GPO |
| **Языки/скрипты** | Bash, YAML (Ansible) |

## 📋 Выполненные задачи

### Часть 1. Сеть (13/13 заданий)

| № | Задание | Результат |
|---|---------|-----------|
| 1 | VLAN на коммутаторах даркстора | Связность внутри VLAN подтверждена ping |
| 2 | Маршрутизация на R-Darkstore | Субинтерфейсы, межсетевая маршрутизация |
| 3 | DHCP в дарксторе | Клиенты получают адреса автоматически |
| 4 | Выход в интернет (даркстор) | Стык с ISP, маршрут по умолчанию |
| 5 | Тестовый интернет-сервер | Доступность 1.1.1.1 с ISP |
| 6 | NAT в дарксторе | PAT для всех VLAN, статистика хитов |
| 7 | VLAN в головном офисе | Связность внутри VLAN 10, 20, 50 |
| 8 | Маршрутизация в HQ | Межсетевая связность VLAN 10↔20↔50 |
| 9 | DHCP в HQ | Автоматическая выдача адресов |
| 10 | Выход в интернет (HQ) | Связь с ISP, маршрут по умолчанию |
| 11 | NAT в HQ | PAT для всех VLAN |
| 12 | Публикация веб-сервера | DNAT настроен (проверка конфигурации) |
| 13 | Связь между локациями | Сквозная маршрутизация через WAN (TTL=61) |

> Подробные отчёты и скриншоты — в [`docs/part1_network/`](docs/part1_network/).

### Часть 2. Сервисы Linux (5/5 заданий)

| № | Сервис | Результат |
|---|--------|-----------|
| 1 | NTP + DNS | Синхронизация времени, разрешение имён в домене `practicumsuperstore.ru` |
| 2 | Ansible | Пользователи созданы, бэкапы по cron доставляются на сервер |
| 3 | Почта (Postfix) | Письма доставляются в Maildir, проверка MX-записи |
| 4 | FTP (vsftpd) | Загрузка файлов работает, локальный пользователь авторизуется |
| 5 | Веб (Nginx) | Страница доступна, лог фиксирует запросы, аутентификация для `/secure` |

> Подробности — в [`docs/part2_services/`](docs/part2_services/).

### Часть 3. Active Directory (1/1 задание)

- Домен `practicumsuperstore.local` развёрнут на Windows Server.
- Создана OU `IT`, пользователь `student`, компьютер `PCI-1`.
- Настроена GPO с приветственным сообщением («Поздравляем / Политика успешно применена!»).
- Клиент введён в домен и применяет политику.

> Скриншоты — в [`docs/part3_ad/`](docs/part3_ad/).

## Мои выводы

В ходе проекта я:
- Углубил понимание **VLAN**, **trunk**, **субинтерфейсов** и статической маршрутизации.
- Освоил настройку **NAT/PAT** и **DNAT** для публикации внутренних сервисов.
- На практике развернул **критические сервисы Linux** (DNS, NTP, почта, FTP, веб).
- Применил **Ansible** для автоматизации рутинных задач (создание пользователей, бэкапы).
- Настроил **Active Directory** с групповыми политиками, что важно для понимания корпоративной безопасности.

Сложности возникли при отладке DNAT (тест telnet не прошёл, но сама конфигурация корректна) и при настройке Ansible для кросс-платформенного взаимодействия, но все проблемы были решены.

Проект полностью соответствует техническому заданию и готов к демонстрации.
