# Инфраструктурный проект для компании «Суперстор»

## О проекте

В рамках учебной проектной работы я спроектировал и настроил корпоративную сеть для компании «Суперстор», которая открывает новый офис и даркстор. Задача включала:

- Построение сети с VLAN-изоляцией отделов (IT, менеджмент, логистика, склад, серверные).
- Настройку маршрутизации, NAT и публикацию веб-сервера в интернет.
- Организацию выделенного канала между двумя площадками.
- Развёртывание базовых и прикладных сервисов: DNS, NTP, почта (Postfix), FTP (vsftpd), веб-сервер (Nginx), система управления конфигурациями (Ansible).
- Установку и настройку Active Directory с применением групповых политик.

**Цель проекта** — продемонстрировать практические навыки сетевого инженера и системного администратора, понимание принципов работы корпоративных ИТ-инфраструктур и их защиту.

## Стек технологий

- **Сетевое оборудование**: маршрутизаторы и коммутаторы Cisco (EVE‑NG)
- **Серверные ОС**: Ubuntu 20.04 LTS, Windows Server
- **Сервисы**:
  - DNS/DHCP: `dnsmasq`
  - NTP: `chrony`
  - Почта: `Postfix` + `mailutils`
  - FTP: `vsftpd`
  - Веб: `Nginx` + аутентификация (htpasswd)
  - Управление: `Ansible`
  - Каталог: `Active Directory Domain Services`
- **Языки/скрипты**: Bash, YAML (Ansible)

## Выполненные задачи

### Часть 1. Дизайн и конфигурация сети (13 заданий)
- Настроены VLAN на коммутаторах, trunk-соединения, субинтерфейсы.
- Запущены DHCP-пулы для пользовательских подсетей.
- Настроен NAT (PAT) для выхода в интернет и DNAT для публикации веб-сервера.
- Организована сквозная маршрутизация между головным офисом и даркстором через WAN.

Все проверки (ping, таблицы маршрутизации, статистика NAT) подтверждены скриншотами — см. [`docs/part1_network/`](docs/part1_network/).

### Часть 2. Настройка базовых и прикладных сервисов (5 заданий)
- Развёрнуты DNS, NTP, почтовый сервер, FTP, веб-сервер.
- Настроен Ansible для управления клиентской машиной (создание пользователя, бэкап по cron).
- Проверена работа всех сервисов (nslookup, timedatectl, отправка почты, загрузка/скачивание по FTP, доступ к веб-странице).

Скриншоты и конфиги — в [`docs/part2_services/`](docs/part2_services/) и [`configs/`](configs/).

### Часть 3. Настройка Active Directory (1 задание)
- Поднят контроллер домена `practicumsuperstore.local`.
- Созданы OU `IT`, пользователь `student`, компьютер `PCI-1`.
- Настроена GPO с приветственным сообщением при входе.
- Клиент введён в домен, политика применена.

Доказательства — в [`docs/part3_ad/`](docs/part3_ad/).

## Выводы

Проект позволил:
- Закрепить навыки работы с VLAN, 802.1Q, субинтерфейсами, NAT и статической маршрутизацией.
- На практике освоить развёртывание популярных сервисов Linux и Windows.
- Применить Ansible для автоматизации.
- Понять принципы работы Active Directory и групповых политик.

В ходе работы возникли сложности с настройкой trunk-портов и отладкой DNAT, но все были успешно решены.
