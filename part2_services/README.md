## Задание 1. Настройка служб NTP и DNS

**Что было сделано:**
- На **Хосте 1** (DNS/NTP-сервер) установлены и настроены `chrony` и `dnsmasq`.
- В `chrony.conf` добавлены:
  - `local stratum 10` — сервер выступает как источник времени для локальной сети.
  - `allow 10.10.0.0/16`, `allow 10.11.0.0/16`, `allow 10.12.0.0/16` — разрешены клиенты из всех подсетей проекта.
  - Временная зона установлена на Москву (UTC+3).
- В `dnsmasq.conf` настроен домен `practicumsuperstore.ru` (в скриншоте есть опечатка, но записи созданы для правильного домена):

  - PTR-записи для обратного разрешения.
  - MX-запись для почтового сервера.
- На **Хосте 2** (Ansible-сервер) настроен клиент NTP и DNS (указан DNS-сервер 10.10.1.86 в `/etc/resolv.conf`).
- Служба `systemd-resolved` отключена (чтобы не конфликтовать с `dnsmasq`).

## Задание 4. Конфигурация FTP-сервера для обмена файлами (vsftpd)

**Что было сделано:**
- На **Хосте 2** установлен и настроен FTP-сервер vsftpd.
- Создан пользователь `ftpuser` с домашней директорией `/home/ftpuser`.
- Создана папка для обмена файлами: `/home/ftpuser/ftp/files`.
- В файле `/etc/vsftpd.conf` включены ключевые параметры:
  - `local_enable=YES` — разрешён вход локальным пользователям.
  - `write_enable=YES` — разрешена запись (загрузка файлов).
  - `chroot_local_user=YES` — ограничение пользователей в их домашней директории.
  - `allow_writeable_chroot=YES` — разрешение записи внутри chroot.
- Служба vsftpd перезапущена.
- На **Хосте 3** (клиент) выполнено подключение к FTP-серверу под пользователем `ftpuser`, загружен тестовый файл (`cat 3.png`) с сервера.
